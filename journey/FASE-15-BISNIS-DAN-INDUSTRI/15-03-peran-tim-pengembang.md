# 15-03: Peran Tim Pengembang

> **Fase**: 15 — Bisnis & Industri  
> **Prasyarat**: 15-02-dari-kode-ke-produk  
> **Waktu baca**: 35-45 menit  
> **Kata kunci**: team, role, developer, PM, designer, QA, DevOps, CTO

---

## 📋 Ringkasan

Aplikasi web tidak dibuat sendirian. Tim pengembang terdiri dari beberapa peran — masing-masing dengan tanggung jawab berbeda.

**Target pemahaman:**
- Kamu paham struktur tim engineering
- Kamu tahu peran masing-masing
- Kamu paham posisi junior dalam tim

---

## 1. Struktur Tim

### 1.1 Tim Kecil (Startup / Agency)

```
CEO / Founder
    │
    ├── Project Manager (PM)
    │
    ├── UI/UX Designer
    │
    └── Developer (Fullstack)
         ├── Junior Developer ← KAMU
         └── Senior Developer

Setiap orang bisa pegang multiple roles.
```

### 1.2 Tim Besar (Enterprise)

```
CTO / VP Engineering
    │
    ├── Engineering Manager
    │   ├── Senior Backend
    │   │   └── Junior Backend ← KAMU
    │   ├── Senior Frontend
    │   │   └── Junior Frontend
    │   └── QA Engineer
    │
    ├── DevOps Engineer
    ├── Product Manager (PM)
    ├── UI/UX Designer
    └── Scrum Master
```

---

## 2. Peran & Tanggung Jawab

| Peran | Tanggung Jawab |
|-------|---------------|
| **PM** | Requirement, prioritas, jembatan client |
| **UI/UX** | Desain tampilan, user flow, prototype |
| **Backend** | API, database, business logic, server |
| **Frontend** | Tampilan, interaktivitas, performa |
| **Fullstack** | Backend + Frontend |
| **QA** | Testing, bug report, regression |
| **DevOps** | Deployment, server, CI/CD, monitoring |
| **CTO** | Arah teknis, arsitektur, tim |

---

## 3. Junior Developer dalam Tim

```text
Tanggung jawab junior:
├── Implementasi fitur (dengan bimbingan)
├── Bug fixing
├── Menulis test
├── Code review (baca PR orang lain)
└── Learning

Harus:
✅ Tanya kalau tidak tahu
✅ Minta code review
✅ Dokumentasi apa yang dipelajari
✅ Estimasi realistis

Jangan:
❌ Diam berlama-lama kalau stuck
❌ Ngasal push tanpa test
❌ Menyalahkan orang lain
```

---

## 4. Cara Kerja Tim

```text
Daily standup (15 menit):
  - Apa yang dikerjakan kemarin?
  - Apa yang dikerjakan hari ini?
  - Ada blocker?

Sprint planning (2 jam, tiap 2 minggu):
  - Tentukan fitur yang akan dikerjakan
  - Estimasi effort

Code review:
  - Setiap PR di-review minimal 1 orang
  - Cari bug, security issue, code style

Retrospective:
  - Akhir sprint: apa yang baik? apa yang perlu diperbaiki?
```

---

## 🧪 Latihan

1. **Identifikasi.** Di perusahaan targetmu, tim terdiri dari peran apa saja? (Lihat LinkedIn / website perusahaan)

2. **Simulasi.** Bayangkan kamu di tim 5 orang (PM, Designer, Senior Dev, Junior Dev = kamu, QA). Bagaimana kamu berinteraksi?

3. **Daily standup.** Tulis apa yang akan kamu katakan di daily standup hari ini untuk codebase ini.
